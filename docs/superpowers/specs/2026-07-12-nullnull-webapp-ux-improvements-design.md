# 널널(Nullnull) 웹앱 UX 개선 설계

작성일: 2026-07-12
대상: `nullnull-travel-webapp`(FE) + `app`(백엔드) 일부
배경: 데모 웹앱 1차 개선 이후 사용자가 지적한 6개 이슈 + VisitKorea(대한민국 구석구석) 참고 UX 친화화.

## 목표

붐빔 분산이라는 핵심 가치는 유지하되, **내부 지표/용어 중심 화면을 방문자 관점의 친화적 UX로** 다듬는다. VisitKorea 메인의 원칙(맞춤 코스 빠른 진입, 카테고리 브라우즈, 실사진 카드, 평이한 언어)을 참고한다.

원칙:
- 공모전 심사(구현성·기획력·데이터 활용·발전성) 서사는 유지하되 사용자 화면에서는 내부 용어(F8·AlternativeScore·로테이션)를 노출하지 않는다.
- 데이터/스코어링의 근거는 남기고, **표현만** 사용자 혜택 언어로 바꾼다.
- 동행은 **하드 필터가 아니라 소프트 우선정렬**로 둔다(사용자 선택 우선).

## 비범위(이번에 하지 않음)

- 카카오맵 전환: 도메인 확정 후 별도 진행(메모리 결정 유지). 이번엔 무료 타일 교체로만 개선.
- 스코어링 가중치 백테스트(9-5)·시드 정책 변경.

## A. 지도 — "스크린샷처럼 뜬다" (#1)

**문제:** `LeafletPointsMap`이 `scrollWheelZoom:false`이고 OSM 기본 타일이 앱 파스텔 톤과 이질적이라 "붙여넣은 정적 이미지"처럼 보인다.

**접근:**
- 타일을 **CARTO Voyager**(`https://{s}.basemaps.cartocdn.com/rastertiles/voyager/{z}/{x}/{y}{r}.png`, 무료·키/도메인 등록 불필요, 밝고 부드러운 톤)로 교체. attribution에 OSM+CARTO 표기.
- **조작 활성화**: 드래그, 더블클릭 줌, 줌 컨트롤(zoomControl) 노출. 휠 줌은 페이지 스크롤 방해 방지를 위해 off 유지하되, `L.map` 옵션으로 명시.
- 마운트 직후 컨테이너 0-크기 → 단색 블록 문제는 기존 `invalidateSize` 워크어라운드 유지. 라운드 클립도 부모 래퍼 유지.

**변경:** `main.jsx`(LeafletPointsMap), 필요 시 `styles.css`(줌 컨트롤 톤).

**수용 기준:** 대안/코스 지도에서 실제 타일이 로드되고, 드래그·줌으로 상호작용되며, 정적 이미지 인상이 사라진다. 실기동 스크린샷으로 확인.

## B. 검색 — "유사한 장소도 검색되게" (#1)

**문제:** `/api/spots` 키워드가 `name`·`overview` substring만 매칭 → 유사·연관 표현이 안 걸린다.

**접근:**
- 매칭 대상을 **name · category_name · addr · tags(JSON) · overview**로 확장. 입력·대상 모두 공백 제거 정규화(예: "서울 숲" ↔ "서울숲").
- 결과가 적을 때(예: < 3), 기준으로 잡힌(또는 최상위) 결과와 **같은 카테고리/인근 장소를 유사 추천으로 보완**해 드롭다운 하단에 "이런 곳은 어때요?" 그룹으로 노출. (연관 API 캐시 `related_spot` 활용 가능)

**변경:** `app/routers/spots.py`(`list_spots`), `main.jsx`(`SpotSearch` 결과 그룹 렌더), `styles.css`(그룹 헤더).

**수용 기준:** "궁", "한옥", "카페" 등 카테고리성 키워드로도 관련 장소가 나오고, 오탈자·띄어쓰기 변형에도 관대해진다.

## C. 홈 정리 — 기준 장소 + 지표 재구성 (#2, #4)

### C-1. '방문 장소' → '기준 장소' (#2, 사용자 결정)
**문제:** 필터의 "방문 장소" 셀렉트는 코스 출발 기준점인데 라벨이 오해를 부른다("방문했다"는 뉘앙스).

**접근:** 라벨을 **"기준 장소"**로, 도움말/placeholder를 "어디부터 시작할까요?"로 변경. 아이콘을 `MapPin`/`Navigation` 계열로. 검색 선택·카드 클릭·셀렉트가 **같은 기준 상태**를 가리킴을 UI로 명확히(선택된 카드에 "기준" 배지).

**변경:** `main.jsx`(HomeScreen FilterControl), `styles.css`.

### C-2. 상단 지표 사용자 혜택 언어로 재구성 (#4, 사용자 결정)
**문제:** "이번 주 평균 혼잡 회피율", "숨은 명소 선택 N회" 등은 내부 집계 용어라 방문자에게 추상적.

**접근:** 값(데이터)은 유지하고 **문구를 사용자 혜택형으로**:
- "이번 주 평균 혼잡 회피율 32%" → **"지금 붐빔 32% 덜한 시간에 추천 중"**
- "이번 주 숨은 명소 선택 1,240회" → **"덜 알려진 곳 1,240곳을 사람들과 나눴어요"** (또는 "덜 알려진 곳 N곳 발견")
- "TourAPI 서울 관광지 N" → **"서울 관광지 N곳에서 골라요"**
- "OO 현재 혼잡도 N%" → 유지(이미 직관적)
- '시드 포함' 고지 문구는 작은 각주로 유지(데이터 정직성).

내부 용어(F8·AlternativeScore·로테이션) 화면 표기는 제거하되, 관리자(#admin) 화면은 그대로 둔다.

**변경:** `main.jsx`(StatCard 문구, ReasonModal/AlternativeCard의 "로테이션 반영" 등 카피).

**수용 기준:** 홈 첫 화면에 내부 지표 용어가 보이지 않고, 각 숫자가 방문자 행동/혜택으로 읽힌다.

## D. 동행 실제 반영 — 소프트 우선정렬 (#3, 사용자 결정 + 피드백)

**문제:** 현재 companion은 체류시간 배율 + 설명 문장에만 반영, 추천 장소 자체는 불변 → "달라지는 게 없다".

**핵심 원칙(사용자 피드백):** 동행은 **하드 필터가 아니라 우선순위 nudge**. 테마 선택(테마 유지)·자유여행 슬롯 카테고리(미식/포토스팟/…)는 사용자의 **명시적 선택으로 그대로 우선**하고, 동행은 그 후보군 **안에서 순서만** 부드럽게 민다. → 혼자여도 '미식' 슬롯을 고르면 맛집이 나오고, 그중 한적/적합한 곳이 위로 온다.

**설계:**
1. `app/scoring/alternative.py`에 `companion_fit(spot, companion, *, low_pctl, tags) -> float | None` 추가.
   - `companion is None` → **None 반환**. `alternative_score`는 weather와 동일하게 None 항을 제외 후 재정규화 → **기존 점수와 완전히 동일(회귀 없음)**.
   - 방향(첫 버전, 가중치·계수는 튜닝 대상):
     - **혼자(solo):** 한적함/숨은 명소 선호 → `0.4 + 0.6 × visitor_low_percentile`
     - **둘이서(couple):** 포토스팟/자연/뷰 선호 → tags에 {포토스팟, 자연} 있으면 1.0, 미식 0.7, 그 외 0.5
     - **가족(family):** 실내·편의 선호 → `0.5 + 0.5 × is_indoor` (공원류 자연 태그 시 +가점, 상한 1.0). 짧은 이동 선호는 기존 mobility 항이 이미 반영.
2. `alternative_score`에 `companion` 항 추가. 가중치는 **작게**(초기 `companion: 0.12`, `weights.yaml`) 두어 theme(0.30)·relief(0.25)가 여전히 지배 → 명시적 테마 선택을 뒤집지 않는 부드러운 nudge.
3. **전파:**
   - `get_alternatives(..., companion=None)` 파라미터 추가 → 후보 스코어에 companion_fit 반영.
   - `/api/spots/{id}/alternatives`에 `companion` 쿼리 추가. FE `fetchSpotContext`/`findAlternatives`가 현재 선택된 companion 전달.
   - 자유여행 코스 슬롯 선정(`course_service`의 recommend 경로)에도 companion_fit을 슬롯 후보 정렬에 반영. 슬롯 카테고리 적합도(`slot_theme_fit`)가 1차, companion_fit이 2차 정렬 키.
4. **가시화:** 코스/대안 화면에 "가족 여행 반영: 실내·도보 적은 곳 우선" 식 **근거 배지**(companion이 있을 때만). 기존 체류시간 배율·설명 문장도 유지.

**변경:** `alternative.py`, `weights.yaml`, `recommend_service.py`(`get_alternatives`), `course_service`(자유여행 슬롯), `app/routers/spots.py`(alternatives 파라미터), `main.jsx`(companion 전달 + 배지).

**리스크/검증:** 백엔드 스코어링 변경이라 회귀 주의.
- companion=None 경로가 기존과 **바이트 동일 점수**임을 테스트로 고정(기존 대안 테스트 스냅샷 재사용).
- companion별로 상위 목록이 **의미대로 이동**하는지(가족→실내 비중↑ 등) 단위 테스트 추가.

**수용 기준:** 동일 기준 장소에서 동행만 바꾸면 대안/코스 상위 장소 순서가 눈에 띄게 달라지고, 그럼에도 사용자가 고른 테마/슬롯 카테고리는 항상 존중된다.

## E. 이미지·가독성 (#5)

**문제 1(무관한 이미지):** 시드 유명지 이미지가 generic placeholder 4장 재사용(경복궁→숲속 사찰, N서울타워→해안길 등).
**문제 2(폴백):** 이미지 없는 TourAPI 장소 209곳이 단일 '해안길' 히어로로 폴백 → 전부 같은 무관 사진.
**문제 3(가독성):** 카드 제목/보조 텍스트 대비·위계, 히어로 위 텍스트 대비.

**접근:**
- 시드 landmark(경복궁·창덕궁·덕수궁·명동거리·홍대거리·북촌·인사동·N서울타워 등) `image_url`을 **실제 visitkorea 이미지 URL로 교체**. `seed_data.py`(소스)와 현재 `nullnull.db`(런타임) 모두 갱신. 매칭되는 TourAPI 항목의 이미지 또는 검증된 CDN URL 사용.
- FE `imageUrl()` 폴백을 단일 사진 → **장소별 결정적 브랜드 그라디언트 플레이스홀더**(spot_id 해시 기반 색 + 카테고리 아이콘 + 이름). "잘못된 사진"이 아니라 "사진 준비 중"으로 읽히게. (SVG/CSS, 외부 요청 없음)
- 카드 텍스트: 제목 대비 상향, 보조 텍스트(`--muted`) 한 단계 진하게, 히어로 오버레이·`text-shadow` 보강.

**변경:** `app/seed_data.py`, `nullnull.db`(스크립트/직접 업데이트), `main.jsx`(`imageUrl` 폴백 컴포넌트), `styles.css`.

**수용 기준:** 경복궁·N서울타워 등 대표 장소가 실제 사진으로 보이고, 이미지 없는 장소는 통일된 자리표시로 보이며, 카드 글씨가 명확히 읽힌다.

## F. 상세 제안칩 하이라이트 박스 (#6)

**문제:** 상세탭의 시간 이동 제안 칩("7월 16(목) 오후엔 '널널'")이 혼잡도 색(`level-N`) 배경을 써서 **초록 하이라이트 박스**처럼 보여 오작동/강조 오해를 준다.

**접근:** `suggestion-chip`의 `level-N` 배경색 제거 → **중립 카드 스타일**(흰 배경 + 얇은 테두리)로 통일하고, 좌측 시계 아이콘 + 우측 화살표로 "탭하면 그 시간으로 이동" 어포던스를 명확히. 혼잡감소%만 작은 컬러 배지로 강조.

**변경:** `main.jsx`(suggestion-chip 클래스), `styles.css`(`.suggestion-chip`).

**수용 기준:** 제안이 "실수로 칠해진 박스"가 아니라 "누르면 이동하는 제안"으로 읽힌다.

## G. 전반 UX 친화화 (VisitKorea 참고)

- **De-jargon:** '널널도'는 브랜드로 유지하되 첫 등장 시 한 줄 설명. '테마 유지율'→'테마 유지 정도', '숨은 명소성' 등 내부 지표 라벨을 사용자 언어로. AlternativeScore 점수 분해 모달은 톤 완화(수치는 유지, 라벨 평이화).
- **정보 위계/여백:** 홈 섹션 순서·간격 정리(VisitKorea식 "맞춤 코스 빠른 진입"을 상단 CTA로 유지).
- **빈 상태 카피** 친근화.

**변경:** `main.jsx`, `styles.css` 카피/스타일.
